\title{RescueBench Agent: A ReAct-Based Emergency Response

\begin{abstract}
We present a comprehensive evaluation of five emergency response allocation
agents across the 20-scenario RescueBench benchmark, spanning four difficulty
tiers. We introduce \textbf{AgentKit}, a code-first architecture in which all
routing, capacity, and capability matching is handled deterministically, with
an LLM (\texttt{gemini-2.5-flash}) consulted \emph{only} for ethical
tie-breaking when two incidents share identical severity and deadline. We
also introduce \emph{Cap-PWRS}, a capability-coverage variant of the
Priority-Weighted Resolution Score that awards partial credit for
multi-vehicle progress. AgentKit achieves zero constraint violations across
all tiers and matches the deterministic planner on all four tiers. A key
finding is that full ReAct with a ValidatorTool \emph{underperforms} Ablated
ReAct on PWRS in three of four tiers: repeated validator rejections increase
step overhead without improving resolution quality. Step efficiency analysis
shows AgentKit achieves a $4\times$ to $9\times$ step advantage over full
ReAct with zero wasted observation or route-verification steps.
\end{abstract}

\section{Introduction}
\label{sec:intro}

Autonomous emergency resource allocation requires constraint-aware planning,
multi-step reasoning, and recovery from unexpected events. Classical heuristics
are brittle to scenario variation, while LLMs without grounding hallucinate
invalid actions such as dispatching unavailable vehicles, ignoring road
restrictions, or assigning patients to full hospitals. The ReAct
framework~\citep{yao2023react} interleaves reasoning traces with tool calls,
grounding the model in world state before it commits to an action.

A central insight motivating this work is that \emph{not all decisions require
an LLM}. Routing, capacity checking, and capability matching are deterministic
operations where code dominates; LLM judgment is valuable only for ethical
tie-breaking when incidents are objectively equivalent on all measurable
dimensions, as in Tier~3. This motivates our main contribution, the
\textbf{AgentKit} architecture.

This paper makes four contributions: (1) a \textbf{five-method evaluation} of
RescueBench across all 20 scenarios and 4 tiers; (2) \textbf{AgentKit}, a
code-first agent reserving LLM calls for ethical tie-breaking with zero
constraint violations; (3) \textbf{Cap-PWRS}, which provides partial credit
for multi-vehicle capability coverage; and (4) a \textbf{step efficiency
analysis} quantifying LLM grounding overhead versus code-based planning.

\section{Related Work}
\label{sec:related}

\paragraph{ReAct and tool-augmented LLMs.}
\citet{yao2023react} demonstrated that synergizing reasoning traces with action
execution improves decision-making benchmarks. Subsequent work applies
tool-augmented LLMs to web navigation~\citep{shinn2023reflexion}, code
execution~\citep{chen2022code}, and tool learning~\citep{schick2023toolformer}.
We extend this line to physical resource allocation under hard constraints.

\paragraph{Emergency response planning.}
Traditional methods use integer programming and greedy
heuristics~\citep{zhang2017ems}; multi-objective formulations balance response
time, capacity, and fairness~\citep{liu2019multi}. LLM-based planners have
not been systematically evaluated in this setting prior to this work.

\paragraph{LLM benchmarks and multi-vehicle dispatch.}
BabyAI~\citep{chevalier2019babyai} and ALFWorld~\citep{shridhar2021alfworld}
test sequential decision-making but lack quantitative constraint-violation
metrics. Cooperative dispatch where no single agent satisfies all task
requirements appears in team formation~\citep{stone2000} and collaborative
robotics~\citep{khamis2015}; Tier~2 of RescueBench instantiates this problem
for LLM agents for the first time.

\section{Agent Design}
\label{sec:agent}

\subsection{Shared Infrastructure}

All five agents share the same simulation. \textbf{WorldState} maintains the
live city graph, vehicle positions, incident status, and dynamic trigger
schedule. \textbf{WorldTool} exposes six API functions for querying map state,
vehicles, incidents, shortest paths, and dispatching or reporting status.
\textbf{ValidatorTool} deterministically verifies every proposed dispatch,
enforcing vehicle availability, capability contribution, capacity, and
vehicle-type-aware Dijkstra routing. The city graph includes per-edge
\texttt{allowed\_vehicle\_types} restrictions; Tier~2 scenarios track
\texttt{covered\_capabilities} per incident, resolving when all required
capabilities are covered by a sequence of vehicles; and Tier~4 encodes timed
infrastructure events (\texttt{dynamic\_triggers}) whose clock advances on
each dispatch.

\subsection{AgentKit Architecture}
\label{sec:agentkit}

AgentKit delegates all mathematics to code; the LLM is consulted only for
ethical ambiguity. It follows a strict observe, plan, act, replan cycle:
\begin{itemize}
  \item \textbf{observe()} polls world state and proactively advances the
    simulation clock every cycle, firing pending dynamic triggers to ensure
    infrastructure failures are detected regardless of dispatch timing.
  \item \textbf{plan()} ranks open incidents by
    $(\text{severity}{\downarrow}, \text{deadline}{\uparrow})$ in pure Python.
    Only when the top two incidents share \emph{identical} severity and
    deadline does it query the LLM with a minimal structured prompt for
    humanitarian ordering.
  \item \textbf{act()} calls \texttt{\_match\_vehicle()} to find the nearest
    idle vehicle contributing at least one uncovered capability, runs the
    ValidatorTool, and dispatches. It falls through to the next incident if no
    vehicle is available, ensuring no dispatches are missed.
  \item \textbf{replan()} flushes the plan queue entirely and rebuilds it from
    fresh world state whenever \texttt{observe()} returns infrastructure alerts,
    enabling correct response to Tier~4 events.
\end{itemize}
The LLM call budget is effectively zero on Tiers~1, 2, and~4.

\subsection{ReAct Architecture}

The ReAct loop grants the LLM full tool access across a multi-turn
conversation, terminating when no further calls are issued or a 40-step
budget is exhausted. \emph{Full} enables ValidatorTool (invalid dispatches
return a structured error for self-correction); \emph{Ablated} disables it
(invalid dispatches execute and corrupt world state).

\section{Experimental Setup}
\label{sec:setup}

\subsection{Benchmark}
RescueBench~\citep{group2bench2026} provides 20 JSON scenarios on a shared
16-node city graph, in four tiers of 5 scenarios each: \textbf{Tier~1}
(basic triage, single-capability incidents, no dynamic events);
\textbf{Tier~2} (multi-capability incidents, restricted edges, fuel
constraints); \textbf{Tier~3} (ethical prioritization under vehicle
scarcity); and \textbf{Tier~4} (infrastructure collapse forcing mid-mission
route recalculation and hospital diversion).

\subsection{Baselines and Metrics}
We compare five methods: \textbf{Deterministic} (greedy sort by
severity/deadline, no LLM, efficiency upper bound); \textbf{Zero-Shot LLM}
(single prompt, JSON dispatch list, no validation); \textbf{ReAct Ablated};
\textbf{ReAct Full}; and \textbf{AgentKit}.

\paragraph{PWRS.}
$\text{PWRS} = \frac{\sum_{i} s_i \cdot \mathbf{1}[\text{resolved}_i \wedge t_i^* \leq d_i]}
                      {\sum_{i} s_i}$,
counting only on-time resolutions weighted by severity $s_i$.

\paragraph{Cap-PWRS.}
\begin{equation}
  \text{cap\_cov}_i = \frac{|C_i^{\text{cov}} \cap C_i^{\text{req}}|}{|C_i^{\text{req}}|}
  \cdot \max\!\left(0,1-\tfrac{t_{\text{res}}-d_i}{d_i}\right),
  \quad
  \text{Cap-PWRS} = \frac{\sum_i s_i \cdot \text{cap\_cov}_i}{\sum_i s_i}
\end{equation}
Awards partial credit for multi-vehicle progress with a linear late penalty.
We also report resolution rate, deadline adherence, violation count, and step
efficiency ($N_{\text{resolved}}/N_{\text{steps}}$). All LLM experiments use
\texttt{gemini-2.5-flash}; each scenario is run once with a 40-step budget.

\paragraph{Tier 1 (Basic Triage).}
Deterministic and AgentKit achieve identical results (PWRS 0.496, Cap-PWRS
0.717, zero violations, step efficiency 1.000). Ablated ReAct leads on PWRS
(0.576) by dispatching more aggressively; full ReAct scores 0.525 with
slightly higher Cap-PWRS (0.744) from wider world-state exploration, but at
$2.4\times$ lower step efficiency.

\paragraph{Tier 2 (Constraint Satisfaction).}
Deterministic and AgentKit match exactly (PWRS 0.400, Cap-PWRS 0.714) with
zero violations. Ablated ReAct and Zero-Shot both achieve higher Cap-PWRS
(0.807) by dispatching more vehicles per incident, but accumulate violations
or lose resolution rate precision. Full ReAct generates 1.4 validator
rejections per run as the model repeatedly attempts dispatches to incidents
where no uncovered capability remains.

\paragraph{Tier 3 (Ethical Prioritization).}
AgentKit exactly matches the Deterministic planner (PWRS 0.410, Cap-PWRS
0.453), confirming the LLM's humanitarian tie-breaking aligned with the
severity-deadline metric. Ablated ReAct and Zero-Shot lead on PWRS (0.460)
by dispatching all vehicles without constraint overhead. Full ReAct accumulates
3.6 validator rejections per run as the trolley-problem structure causes
repeated retries of blocked vehicles.

\paragraph{Tier 4 (Dynamic Replanning).}
Zero-Shot and Ablated ReAct both achieve PWRS 0.506, outperforming
Deterministic and AgentKit (0.306) and Full ReAct (0.306). Zero-Shot's global
upfront planning captures replanning opportunities that step-by-step methods
miss because the model encodes the full trigger schedule in its single
response. AgentKit matches Deterministic because \texttt{replan()} correctly
rebuilds the plan queue after each trigger fires, but vehicle coverage is
bounded by available fleet after re-routing.

\paragraph{Ablation and step efficiency.}
Adding the ValidatorTool \emph{hurts} PWRS in three of four tiers (Full vs.
Ablated: mean PWRS 0.410 vs.\ 0.486). The validator catches 0 to 3.6 illegal
attempts per run, but the correction feedback loop increases total steps and
causes the model to over-deliberate on constrained decisions. AgentKit achieves
the same zero-violation guarantee as Deterministic with $4\times$ to $9\times$
better step efficiency than Full ReAct because all routing is performed by
vehicle-type-aware Dijkstra inside \texttt{\_match\_vehicle()} at zero LLM
cost. Of the ReAct loop overhead, approximately 42\% of steps are initial
observation calls needed to ground the LLM, 29\% are route-verification
\texttt{get\_shortest\_path} calls, and only 29\% are productive dispatches.
For AgentKit, 100\% of steps are dispatches.

\section{Discussion}
\label{sec:discussion}

\textbf{Code-first design eliminates constraint violations.}
AgentKit achieves zero violations across all 20 scenarios because every
dispatch passes through \texttt{\_match\_vehicle()} (enforcing capability
contribution, capacity, and route reachability in code) and then through the
ValidatorTool before any world state mutation. By advancing the simulation
clock on every observation cycle, AgentKit also detects Tier~4 infrastructure
failures immediately rather than only on dispatch arrival, ensuring
\texttt{replan()} fires as soon as a bridge collapses or hospital floods.
Cap-PWRS reveals substantial multi-vehicle partial progress on Tier~2
incidents (Cap-PWRS significantly exceeds binary PWRS for all methods),
confirming that vehicles are covering individual capabilities even when full
resolution is not achieved within the step budget.

\vspace{0.25em}
\textbf{ValidatorTool correction loops are costly.}
The full ReAct loop achieves lower mean PWRS (0.410) than Ablated ReAct
(0.486). The validator correctly blocks illegal dispatches (1.8 rejections/run
average), but the structured error feedback causes the model to repeatedly
retry rejected vehicles rather than moving to lower-priority incidents.
Validator-augmented agents likely require explicit retry budgets or
negative-example injection to avoid exhausting the step budget in correction
cycles.

\vspace{0.25em}
\textbf{Limitations and failure modes.}
Tier~4 resolution is bounded by post-collapse vehicle availability (PWRS
0.306 for both Deterministic and AgentKit), not a planning deficit. The LLM
ethical tie-breaking feature is difficult to evaluate because the LLM's
priorities coincided with the quantitative metric, making it impossible to
distinguish a good LLM call from mere agreement. All LLM experiments use a
single model family (\texttt{gemini-2.5-flash}); run-to-run variance is
unquantified since each scenario is run once; and the benchmark covers 20
scenarios on a fixed 16-node graph rather than real-world scale.

\section{Conclusion}
\label{sec:conclusion}

We evaluated five emergency response agents across the full RescueBench
benchmark (20 scenarios, 4 tiers), introducing the AgentKit architecture, the
Cap-PWRS metric, and a step efficiency analysis. AgentKit achieves zero
constraint violations and matches the Deterministic planner on all four tiers,
demonstrating that code-first design produces deterministic-grade reliability
with $4\times$ to $9\times$ better step efficiency than full ReAct. The
ValidatorTool, while correctly blocking illegal actions, reduces PWRS in a
step-limited ReAct loop by inducing correction cycles, suggesting that
tool-guarded agents require explicit self-correction budgets or
negative-example injection. Future work should explore retry budgets for
validator-augmented ReAct and evaluation of AgentKit's ethical reasoning
across a broader set of LLM providers to determine when LLM humanitarian
priorities diverge from severity-based metrics.
